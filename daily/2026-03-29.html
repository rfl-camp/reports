Linus Torvalds 合并了 Miguel Ojeda 维护的 rust-7.1 标签，这是 Rust 在 Linux 内核中的又一次重要推进。这次合并包含了大量工具链和基础设施更新：

简单来说，这就像 Linux 内核的"软件商店"接受了一批新的 Rust 语言"货物"——以后用 Rust 写内核代码会更方便、更安全。同时，Linux 7.0 正式发布（commit 028ef9c9），标志着一个开发周期的结束和新周期的开始。

此外，本次合并窗口还合入了大量 VFS（虚拟文件系统）子系统的更新，包括文件完整性校验支持、目录操作接口简化、扩展属性（xattr）基础设施重构、以及回写（writeback）辅助 API 的引入。这些虽然是 C 代码的改进，但它们改善了内核基础设施，未来 Rust 封装也会受益。

🔧 GA100 GPU 全面支持

Nova-Core 是 NVIDIA 开源 GPU 驱动的 Rust 实现，今天是它的"高产日"，一共有十多个补丁在推进。我们按功能分组来看：

GA100 是 NVIDIA 的一款高端计算 GPU（没有显示输出，专门用来做科学计算和 AI 训练的那种）。这次一共有 4 个补丁让 Nova-Core 支持 GA100：

📬 GA100 固件签名补丁 📬 frts_size() 补丁
📄 统一页表项抽象（v11，20 个补丁）

这是一个大型补丁系列，由 Joel Fernandes (NVIDIA) 提交，Gary Guo、John Hubbard、Alexandre Courbot 等人参与审核，共收到 19 条 review 意见。它做的事情是：为 Nova-Core 的 MMU v2 和 v3 两种页表格式创建统一的包装类型（Pte、Pde、DualPde）。

打个比方：GPU 有两种不同版本的"地址翻译手册"（MMU v2 和 v3），以前代码需要到处写 if v2 ... else v3 ... 来区分。现在用 Rust 的枚举（enum）把两种格式统一封装了，上层代码只需要调用统一接口，不用关心底层是哪个版本。这让页表遍历器（page table walker）和虚拟内存管理器（VMM）的代码变得更干净。

🧹 SysmemFlush 资源清理修复

Danilo Krummrich 提交了一个修复：当 GPU 初始化过程中如果在 SysmemFlush 创建之后发生了其他错误，之前的代码不会自动清理 SysmemFlush 资源（因为它需要手动注销）。现在给它加了 Drop 实现——这是 Rust 的 RAII 模式，当对象被丢弃时自动执行清理操作。同时保留了手动注销路径，因为手动路径可以保持不会失败（infallible），而 Drop 路径依赖运行时状态可能失败。

ACPI 设备驱动修复

Markus Probst 提交了一个重要的 bug 修复（v3），解决了 Rust 驱动在 ACPI PRP0001 设备上匹配数据丢失的问题。

ACPI 是电脑硬件的"自我介绍系统"，操作系统通过它来发现和配置硬件。PRP0001 是一种特殊的 ACPI 设备标识，用于让 ACPI 设备兼容设备树（DeviceTree）的匹配方式。问题出在 Rust 驱动使用了 device_get_match_data 来获取匹配信息，但 Rust 在设备 ID 表中存储的是索引而不是数据指针，导致在 PRP0001 设备上 id_info 总是返回 None。

修复方法是：导出内核的 acpi_of_match_device 函数，让 Rust 代码直接用它来匹配 of 设备表和 ACPI PRP0001。Rafael J. Wysocki（ACPI 子系统维护者）和 Gary Guo 共 8 人参与了审核。
