Tyr 驱动全面采用 register! 宏（v4，共 6 个补丁）

Tyr 驱动全面采用 register! 宏（v4，共 6 个补丁）

Tyr 是一个用 Rust 编写的 ARM Mali GPU 驱动（想象一下手机和平板里的显卡芯片）。在过去的代码里，开发者要和硬件"说话"时，需要手动做大量的位操作——就是把一个 32 位的数字按位拆开、拼装，来读写硬件寄存器。这种操作非常容易出错，比如搞错某个位的位置、忘记掩码等。

这次 Deborah Brouwer 提交的 v4 系列补丁，将 Tyr 驱动中所有硬件寄存器的访问方式，全部换成了内核提供的 register! 宏。这个宏会自动生成类型安全的寄存器和字段访问器，开发者只需要声明"这个寄存器有哪些字段，每个字段占几位"，宏就会帮你生成正确的读写代码。Boris Brezillon 已经对其中的 JOB_CONTROL 部分进行了审查。

解决 Clippy 1.88.0 的格式化参数警告（已合入 rust-next）

Rust 的代码检查工具 Clippy 在 1.88.0 版本引入了一条新规则 uninlined_format_args，它建议把 format!("{}", x) 简写成 format!("{x}")。但这条规则在内核的宏代码中产生了误报警告。Miguel Ojeda 和 Tamir Duberstein 合作提交了两个补丁来解决这个问题：第一个补丁在 kbuild 中允许（即不报告）这条 Clippy 规则；第二个补丁把 rust/macros/module.rs 中确实可以简化的 format! 调用改成了新写法。这两个补丁已经合入 rust-next 分支。

修复 "multiple candidates for rmeta dependency core" 编译错误（V8）

Mukesh Kumar Chaurasiya（IBM）提交了第八版修复补丁，解决一个影响 PowerPC 和其他架构的编译问题。当使用 LLVM=1 并且以单线程（-j1）构建时，rustc 会发现系统上有两份 core 库——一份是系统工具链自带的，另一份是内核自己编译的 libcore.rmeta——于是报错说"找到了多个候选"。这个补丁通过调整搜索路径来消除歧义，确保 rustc 只看到内核自己编译的那份。这个问题在并行编译时不容易出现，但在 -j1 调试构建时会频繁遇到。

时钟子系统实现 Send 和 Sync（已合入 rust-next）

时钟子系统实现 Send 和 Sync（已合入 rust-next）

Alice Ryhl 为 kernel::clk::Clk 类型实现了 Send 和 Sync 两个 trait。在 Linux 内核中，时钟子系统（clk）负责管理硬件的时钟信号——CPU、GPU、外设都需要时钟才能工作，就像我们工作需要看钟表一样。驱动程序通常会把 Clk 嵌入到自己的数据结构中，而这些数据结构需要在线程间传递（Send）。由于 Clk 底层的 C 接口本身是线程安全的，所以为它标记 Send + Sync 是正确的。这个改动使得 Tyr 和 th1520 PWM 等驱动不再需要手动标注线程安全，代码更简洁了。

工作队列：添加安全的取消和状态查询方法（v3）

工作队列：添加安全的取消和状态查询方法（v3）

Aakash Bollineni 提交了工作队列（workqueue）基础设施现代化的第三版补丁系列，共 3 个补丁。工作队列是内核中"延迟执行任务"的机制——当你不想在当前上下文中做某件事（比如中断处理中不能做太耗时的操作），就把它丢到队列里，让内核工作线程稍后执行。这个补丁系列增加了安全的取消和状态查询方法，并且修复了 v2 中一个关键 bug：DelayedWork（延迟工作）的 C 辅助函数没有正确注册定时器回调函数，导致 KUnit 测试崩溃。现在这个问题已经修复，Arc 引用计数的管理也做了整合优化。

通用内存屏障抽象

Joel Fernandes 提交了 3 个补丁，为 Rust 内核代码提供通用的内存屏障抽象。内存屏障是多核处理器编程中的一个重要概念：现代 CPU 为了提高性能，可能会乱序执行指令、乱序写入内存。内存屏障就是告诉 CPU"到这里必须保证之前的操作都完成了"，就像排队时的隔离栏一样。Gary Guo 参与了代码审查，讨论了如何正确地把 C 内核中的 smp_mb()、smp_rmb()、smp_wmb() 等接口在 Rust 中安全地封装。

添加 const_align_up() 函数（v9，已合入 rust-next）

添加 const_align_up() 函数（v9，已合入 rust-next）

John Hubbard 提交了 const_align_up() 函数，这是一个可以在编译期（const context）使用的地址对齐工具函数。什么是地址对齐？想象一下书架上的书，如果每本书都从格子的起始位置放起会更整齐——内存也一样，很多硬件要求数据存放在特定倍数的地址上（比如 4 字节对齐、页面对齐）。const_align_up() 把一个地址向上取整到指定的对齐边界。这个补丁已经来到了第九版，说明社区对接口设计讨论得非常充分，现已合入 rust-next。Miguel Ojeda 参与了审查。
