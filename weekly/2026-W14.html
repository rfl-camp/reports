📋 今日概览

本周是 Rust-for-Linux 项目的里程碑式一周。Rust 最低支持版本正式提升至 1.85.0，与 Debian Trixie 保持一致，这意味着项目可以使用更多稳定的 Rust 特性，大量的代码简化和清理工作随之展开。NVIDIA 的 Nova-Core GPU 驱动继续高速迭代，v9 和 v10 版本带来了内存管理、GSP 固件通信修复和 fwctl 控制接口等重要更新。PowerPC 架构的 Rust 支持补丁从 V7 快速迭代到 V9，解决了关键的编译竞争条件。令人兴奋的是，时隔十年，bus1 —— 一个基于能力的 IPC 系统以 Rust 实现的形式重新出现在社区面前。此外，LED 子系统抽象、工作队列安全取消机制、内存屏障通用接口、网络设备抽象等多个方向都有实质性进展。drm-rust-next 分支已关闭新特性提交，准备进入 v7.1 发布周期。

RUST 工具链与版本升级

Rust 工具链与版本升级

本周最重要的变化之一：Rust 最低支持版本（MSRV）从 1.78.0 提升到了 1.85.0。这个决定由 Miguel Ojeda 提出，依据是去年 LPC 2025 和维护者峰会上达成的共识——跟随 Debian Stable（Trixie 版本）的 Rust 版本。这一升级引发了一系列"连锁反应"式的代码清理，收到了 Benno Lossin、Tamir Duberstein、Gary Guo 等 8 位核心开发者的审阅。

这个升级意味着什么？ 举个例子，之前内核代码需要用不稳定的 DispatchFromDyn 和 Unsize trait 来做智能指针转换，现在可以换成稳定的 CoercePointee 宏了。还有 NonNull::add()、extract_if 等 API 也变得可用，代码变得更简洁。

📬 查看 MSRV 升级补丁

版本升级后，Gary Guo 和 Miguel Ojeda 迅速跟进了一批简化补丁：移除不再需要的 RUSTC_HAS_COERCE_POINTEE Kconfig 选项、简化 --remap-path-prefix 变通方案、清理 Clippy 的 MSRV 配置、移除 nova-core bindings 中不必要的 cfg_attr 等。这就像搬进新房后把旧家具扔掉一样，让代码库更整洁。

📬 简化 NonNull::add()

Clippy 格式化参数警告：Rust 1.88.0 的 Clippy 引入了 uninlined_format_args 警告（建议把 format!("{x}", x=val) 写成 format!("{val}")）。由于内核的宏生成代码暂时无法适配，Miguel Ojeda 先添加了 allow 规则抑制这个警告，同时简化了 macros 模块中的 format 参数写法。

MSRV（Minimum Supported Rust Version）：最低支持的 Rust 编译器版本。就像手机 App 要求最低 iOS 版本一样，内核的 Rust 代码也要指定最低支持哪个版本的 Rust 编译器。版本越新，能用的语言特性越多，但也要考虑用户能否方便地获取该版本。跟随 Debian Stable 是一个务实的选择。

️ GPU/DRM 驱动（Nova-Core 与 Tyr）

Nova-Core v9 大型补丁集（31 个补丁）持续推进。这是 NVIDIA 用 Rust 编写的开源 GPU 驱动，本周涵盖多个子系统的更新：

内存管理 v10：Joel Fernandes 发布了第 10 版内存管理支持补丁（21 个子补丁），包括 TLB 刷新（Translation Lookaside Buffer flush）支持，这是 GPU 虚拟内存管理的关键操作——当页表映射改变时，必须通知硬件清除缓存的地址转换记录。Eliot Courtney 和 Matthew Brost 参与了审阅。

📬 TLB 刷新

GSP 固件通信修复：Alexandre Courbot 修复了命令队列代码中的未定义行为（UB）问题，经历了 v3 到 v5 的快速迭代。问题在于 driver_read_area 和 driver_write_area 返回的切片虽然值正确，但在某些条件下创建方式不安全。John Hubbard 甚至提交了一个 revert 补丁，因为某个修复中使用了 ptr::project!() 对运行时值做了不可靠的范围索引，当编译器无法证明索引在界内时，build_error!() 回退会导致编译失败。这场"修复→revert→再修复"的过程展示了 Rust 在内核中保证安全性的严格要求。

📬 Revert 补丁

内存屏障修复：Gary Guo 发现 GSP→CPU 消息传递路径中缺少读屏障，并将现有屏障更新为 DMA 屏障（带释放语义）。他还提交了一个通用的内存屏障接口，支持完整系统/DMA/SMP 三种作用域和 Read/Write/Full 三种排序方式，让驱动开发者必须明确声明屏障的意图。这是从 nova-core 的实际需求中抽象出的通用基础设施。

📬 通用内存屏障接口

fwctl 固件控制接口：Joel Fernandes 为 nova-core 添加了 fwctl 驱动，提供固件控制接口。Jason Gunthorpe 和 Gary Guo 参与讨论，共 4 条审阅意见。这让用户空间工具可以与 GPU 固件进行交互。

📬 查看 fwctl

bitfield Default 实现修复：Alexandre Courbot 发现 nova-core 的 bitfield 宏生成的 Default 实现并没有真正设置字段的默认值（相当于一个"空壳"默认值）。这个 bug 在 v1 到 v2 迭代中被修复并添加了测试。Danilo Krummrich、Gary Guo、Eliot Courtney 参与审阅。

📬 bitfield 修复 v2

Hopper/Blackwell 启动优化：John Hubbard 的补丁允许在 Hopper 和 Blackwell 架构 GPU 上跳过 GFW 启动等待，并添加了 FSP Chain of Trust 启动支持。另外还有 DMA 掩码宽度设置、固件签名段查找、Falcon DMA 对象对齐等改进，收到了 Alexandre Courbot 等人的审阅。

️ 构建系统与编译优化

C helper 内联到 Rust（无需 Full LTO）v2：这是本周最具技术深度的话题之一。Gary Guo 提交的实验性 Kconfig 选项 RUST_INLINE_HELPERS 已合入 rust-next。Linux 内核中很多 C 函数是 static inline（内联函数）或宏，Rust 无法直接调用它们，需要用 C "helper" 函数做中间桥梁。但这些 helper 不会被内联优化，导致额外的函数调用开销。之前的解决方案需要 Full LTO（全程序链接时优化），编译极慢。新方案在不需要 Full LTO 的情况下实现了 helper 内联，Christian Schrefl、Arnd Bergmann、Geert Uytterhoeven 等人热烈讨论了跨架构兼容性问题。

PowerPC 架构 Rust 支持

Mukesh Kumar Chaurasiya（IBM）本周推动 PowerPC Rust 支持补丁从 V7 快速迭代到 V9，展现了惊人的开发节奏。整个系列包含 4 个子补丁，支持 ppc64le（小端 64 位）和 ppc32be（大端 32 位）两种变体。

关键修复——Makefile 竞争条件：在使用 -j1（单线程编译）加 LLVM=1 时，rustc 会同时发现工具链自带的 libcore.rmeta 和内核本地编译的 libcore.rmeta，导致"多个候选 rmeta 依赖"的错误。修复方案是让 libproc_macro2、libquote 和 libsyn 正确依赖工具链提供的 libcore.rmeta。Gary Guo 和 Miguel Ojeda 参与审阅。

📬 竞争条件修复 V9

ppc32 支持：由 Link Mauve 贡献，最初在 Nintendo Wii 上测试，计划扩展到 GameCube、Wii U 和 Apple G4。ppc64le 在 powernv9 上验证通过，成功加载了 rust_minimal 示例模块。

BUS1：基于能力的 IPC 系统

bus1：基于能力的 IPC 系统

本周最令人兴奋的 RFC 之一：bus1 —— 一个用 Rust 重写的基于能力（capability-based）的进程间通信系统，由 David Rheinsberg 提交了 16 个补丁的完整系列。正如作者所说："十年过去了，想法还是一样的。" bus1 的概念可以追溯到 2016 年，如今终于以 Rust 实现的形式回归。

这个 RFC 包含了丰富的基础设施：LockedBy::access_mut_unchecked()（无需可变借用即可访问）、Vec::into_boxed_slice()（消耗 Vec 的分配转为 boxed slice）、基本工具函数、字段投影（field projections）、IntoDeref/FromDeref trait、侵入式红黑树等。Miguel Ojeda 对多个补丁进行了细致的审阅。

值得注意的是，字段投影模块为稳定 Rust 提供了类似官方 field projections RFC 的功能，允许安全地从一个引用"投射"到其内部字段的引用——这在内核中操作复杂数据结构时非常有用。

Jonathan Corbet 也参与了关于 man-page 补丁的讨论。这个系列展示了 Rust 在实现复杂内核子系统时的潜力，也为内核社区提供了大量可复用的 Rust 基础组件。

📬 bus1 RFC 总览
💡 小白科普
IPC（Inter-Process Communication）：进程间通信。操作系统中不同程序之间需要交换数据和消息，IPC 就是这个通道。Linux 已有管道、共享内存、Socket 等 IPC 机制，bus1 基于"能力"模型，每个消息端点拥有明确的权限，类似于"持有钥匙才能开门"的安全模型。
字段投影（Field Projection）：安全地从一个结构体的引用获取其内部字段的引用。在内核中，你经常需要从一个大结构体"深入"到某个字段来操作，字段投影确保这个操作在 Rust 的所有权和生命周期系统下是安全的。

SPDM 安全协议

rspdm RFC v3 持续讨论：Alistair Francis 的 rspdm（Rust SPDM 实现）RFC 系列中的证书获取和证书验证两个补丁引发了讨论。Gary Guo 和 Miguel Ojeda 参与了评审。SPDM（Security Protocol and Data Model）用于设备之间的安全认证和通信，在 CXL、NVMe 等现代硬件接口中越来越重要。用 Rust 实现这类安全关键协议可以减少内存安全漏洞。

📬 SPDM 证书验证
💡 小白科普
SPDM：Security Protocol and Data Model，由 DMTF 组织定义的安全协议标准。它让计算机能够验证硬件设备的身份和完整性——确保你插入的 PCIe 设备确实是它声称的那个设备，而不是被篡改的恶意硬件。
